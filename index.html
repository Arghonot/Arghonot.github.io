My_CustomBehaviorTree

My_CustomBehaviorTree is my try at coding a behavior tree.

It started with my other project Battelefront, I needed a tool that allowed me to quickly design an agent's behaviour.

I had numerous attempts in order to do so, the classic MonoBehavior AI, scriptable AI, finite state machine and finally 3D parties assets like Behavior bricks. The lack of documentation on Behavior Bricks made me do my own try at coding a behavior tree.

The idea is simple, you design your agent's behaviour by linking nodes together. Each node is a simple task that the agent might or might not do. On the picture above you can see the first version of my agressive behavior for battlefront.

Battlefront

Battlefront is a side project I had been wanting to work on and finish for a long time now.

The setup of the game is two teams of AI which are battling each other in order to gain complete control over the map.

The two teams are composed of the same amount of AIs, each of them will have different classes.
These classes define the AI's equipment as follow :
  • Assault rifle
  • Shotgun
  • Rocket launcher
  • Sniper rifle
  • Grenade launcher

Each AI will have to work on their own tactic in order to beat the other team while trying to stay alive.
Each map contains a control point that each AI will have to capture in order to gain a respawn point.
A game ends when a team has no more spawn tickets left or when all the command posts are occupied by one team.

Controller

Party hard is a mario-party like game.

The idea is to play through a quick succession of short levels with one, two or three other players on a good old split-screen game.

To me this project is more like a game design laboratory, the very fast pace will allow me to focus on one simple idea per level.

For now this project mainly contain the frame of the final game that I will build level upon. Yet there are some test levels like one with falling platform or another one with falling bricks that all the players have to avoid.

I intend to work on this game again once I finish my other project, battlefront.

On a technical standpoint this project allowed me to do a lot of iteration on a "generic level" based game and to test a lot of different game architecture for an event driven game.
