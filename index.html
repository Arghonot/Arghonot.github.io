Dragonfly

3 years

DragonFly is a multi-platform industrial data visualization and manipulation tool (Desktop + VR) developed in Unity by a team of a dozen developers within Airbus Defence & Space. Chosen as the group’s official VR visualizer, it has received multiple internal awards and is used across various departments.

I was part of DragonFly’s core development team for three years, contributing both as a feature developer and integration lead. I worked on key modules including measurement tools, 3D gizmos, and custom API integrations (TCP/UDP/REST), as well as companion tools like a desktop bug reporter.
My primary focus, however, was collaborating with other internal teams to adapt and deploy DragonFly into their operational workflows — translating their needs into features, aligning roadmaps, and ensuring successful production rollouts.

  • Visual studio / Unity / Blender / Git / Jira / SQLite

Space simulation

10 months

I was tasked with building a proof of concept — and later, a full implementation — of a real-time solar system simulation to enable DragonFly to visualize satellite missions in both Desktop and VR. The goal was to integrate DragonFly with external mission simulation tools like VTS and Celestia, allowing users to observe satellite telemetry and trajectories in an immersive, interactive environment.

To achieve this, I developed a custom double-precision coordinate system to overcome Unity’s floating-point limitations at astronomical scales, ensuring accurate orbital positions and seamless motion. I also helped on the implementation of a dynamic lighting model and rendering pipeline for the solar system and built the interface layer to receive and parse real-time data from external systems.

This project helped in demonstrating DragonFly’s potential as a mission analysis tool and was deployed in a VR setting at Toulouse Mission Control for internal use.

Technologies

  • C# / C
  • Visual studio / Unity / Visual studio 2007 / VTS / Celestia / Blender / Git
  • TCP / XML

The Farting Game

Game Jam 2024

Welcome to The Farting Game, a chaotic 2–4 player local co-op game set in a crowded office elevator full of NPCs. Your mission: unleash your gas strategically while staying hidden — and sniff out the other players before they get you.

But beware: players can cork each other's gas pipes to disrupt plans or punch when they suspect foul play. It’s absurd, competitive, and surprisingly tactical.

My responsibilities:
  • Took part in design and gameplay discussions during the concept phase.
  • Wrote a prototype for the NPC steering and crowd movement system.
  • Sourced and selected all graphic assets for the game.
  • Developed all in-game custom shaders and visual effects.
  • Established and implemented the overall artistic direction.
  • Unity
  • Shader Graph, custom stylized rendering
  • Local multiplayer (keyboard/gamepad)
